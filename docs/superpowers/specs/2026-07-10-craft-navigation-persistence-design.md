# 写作模式导航与最近选择持久化设计

## 目标

调整写作模式页面的导航结构，并让最近选择的写作模式持久化到项目数据库：

- 顶部导航栏始终显示“模式列表”“新建模式”“模式详情”。
- “模式详情”不再只在进入详情后临时出现。
- 用户选择或新建模式后，将该模式记录为项目最近模式。
- 重新进入写作模式页面时，自动恢复最近模式并显示详情。
- 删除最近模式时，自动清理无效记录并切换到可用模式或列表空状态。

## 当前问题

`CraftManager` 当前把 `tab`、`selectedCraftId` 和新建结果保存在组件内存中：

- 详情标签只有在 `tab === "detail"` 时渲染。
- 刷新或离开页面后，最近选择丢失。
- 删除当前模式后没有统一的最近模式回退策略。
- 选择状态和写作模式数据没有明确的持久化边界。

## 方案与边界

采用项目级 SQLite 数据库，不把界面状态写入 `inkos.json` 或浏览器 `localStorage`。

- 数据库路径：`<projectRoot>/.inkos/studio.db`。
- `.inkos/` 是项目运行时数据目录，已经由项目规则加入忽略范围。
- 数据库仅保存最近模式 ID，不复制写作模式内容；模式正文仍由现有 `crafts/<id>/craft_profile.json` 管理。
- 读取或写入偏好失败时，页面仍可使用模式列表，不阻塞模式详情展示；失败应记录服务端日志。

## 数据库结构

新增一个轻量偏好表，支持未来扩展其他 Studio 项目级偏好：

```sql
CREATE TABLE IF NOT EXISTS studio_preferences (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

本功能使用的键为 `recent_craft_id`，值为现有写作模式 ID。

数据库访问封装在 Studio API 层，初始化时创建目录、数据库和表；每次请求使用短生命周期连接并显式关闭，避免 Windows 下文件句柄阻塞 worktree 或运行时目录清理。

## API 设计

### 获取模式列表

扩展现有 `GET /api/v1/crafts` 响应：

```json
{
  "crafts": [],
  "recentCraftId": "craft-id-or-null"
}
```

如果数据库读取失败，`recentCraftId` 返回 `null`，不影响列表返回。

### 保存最近模式

新增 `PUT /api/v1/crafts/recent`：

```json
{ "craftId": "craft-id" }
```

服务端必须确认该模式存在后再写入；不存在时返回 404，避免保存悬空 ID。

### 清除最近模式

新增 `DELETE /api/v1/crafts/recent`，用于无模式或删除最近模式后的显式清理。

### 删除模式联动

现有 `DELETE /api/v1/crafts/:id` 在删除模式后检查 `recent_craft_id`：

- 如果被删 ID 是最近模式，清除该记录。
- 否则保留现有最近模式。

`/crafts/recent` 路由必须在 `/crafts/:id` 之前声明，避免被动态 ID 路由误匹配。

## 页面状态与交互

### 状态模型

保留现有三种页面状态：`list`、`create`、`detail`，但导航栏始终渲染三个标签。新增 API 返回的 `recentCraftId` 作为初始选择来源：

1. 首次加载等待模式列表和最近模式查询完成。
2. 最近模式存在且仍在列表中时，设置 `selectedCraftId` 并默认进入 `detail`。
3. 最近模式为空或已失效时，默认进入 `list`。
4. 用户点击列表项时保存最近模式并进入 `detail`。
5. 新建成功时保存新模式并进入 `detail`。
6. 用户点击“模式详情”时显示当前最近模式；没有选择时显示空状态并提供进入列表/新建的操作。

### 导航栏

- 放在内容区域顶部，使用全宽、统一间距的导航栏样式，而不是当前的窄内容宽度下的临时 tab 条。
- 三个导航项始终可见。
- 当前页面使用底部边框和主题色高亮。
- “模式详情”没有最近模式时保持可点击，但显示空状态，不制造隐藏或失效导航项。
- 列表中的最近模式增加选中高亮，详情页标题与内容始终对应同一个 ID。

### 回退规则

- 最近模式被删除：清除数据库记录，按创建时间选择最新可用模式；若没有模式则进入列表空状态。
- 最近模式读取失败：默认列表模式，后续选择仍可以尝试写入。
- 详情加载失败：保留最近模式 ID，显示错误/空状态，不自动改写数据库；用户可回到列表重新选择。
- 新建模式成功后：新模式成为最近模式，不再回到列表。

## 测试设计

### API 测试

- 首次读取无记录返回 `recentCraftId: null`。
- 保存存在的模式后可重新读取。
- 保存不存在的 ID 返回 404，数据库不写入悬空值。
- 删除最近模式后记录被清除。
- 删除非最近模式不影响最近模式。
- 数据库初始化可重复执行，表结构不会重复创建。

### 页面测试

- 三个导航标签在列表、创建、详情三种状态下都存在。
- 列表点击模式后保存选择并切换详情。
- 有最近模式时初始化进入详情并显示对应模式。
- 无最近模式时详情显示空状态。
- 删除当前模式后按回退规则切换或回列表。
- 最近模式在模式列表中有高亮状态。

### 运行验证

- Studio 类型检查。
- Studio API 和 CraftManager 相关 Vitest 测试。
- 核心包构建，确认 `node:sqlite` 只在服务端代码路径使用。
- 启动 4567/4569，刷新浏览器并验证列表、创建、详情和删除后的状态恢复。

## 非目标

- 不改变写作模式解析内容和模型调用逻辑。
- 不把完整写作模式 profile 迁移到 SQLite。
- 不为最近模式增加跨项目同步或账号级同步。
- 不改变左侧全局导航和其他页面的路由结构。
