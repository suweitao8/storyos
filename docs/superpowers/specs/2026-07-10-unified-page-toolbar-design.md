# 统一页面导航栏设计

## 背景

当前 Studio 的中间内容区只有细分隔线，语言和主题切换在设置页中，写作模式和导入页面分别实现自己的 tab。用户希望把常用交互集中到右侧页面顶部，并让所有页面使用统一的导航视觉。

## 目标

- 所有右侧主内容页顶部都有一致的页面导航容器。
- 语言切换和主题切换在全局导航中可直接使用。
- 页面级 tab、返回、刷新、新建、保存等操作使用相同的视觉和间距规范。
- 保留现有路由、状态和后端接口，不引入重复的全局状态。
- 桌面端保持紧凑，窄屏下导航内容可以横向滚动，不挤压正文。

## 非目标

- 不重做左侧 Sidebar。
- 不改变页面业务流程和路由模型。
- 不新增设置或导航相关后端接口。
- 不把所有页面的业务操作强行塞进全局导航，页面独有操作仍由页面提供。

## 方案

新增共享 `PageToolbar` 组件，提供统一的页面级导航容器：

- `title`：页面标题或当前上下文。
- `tabs`：可选的页面 tab 列表和激活状态。
- `leading`：返回、面包屑等左侧操作。
- `actions`：刷新、新建、保存等页面操作。
- `globalActions`：应用级语言和主题切换。

App 在中间内容区统一渲染全局 toolbar，并根据当前路由生成页面标题。需要业务 tab 的页面在自己的内容区域紧接着渲染 `PageToolbar` 的 tab 变体，避免 App 需要持有页面内部状态。语言切换沿用现有 `PUT /project` 和 `refetchProject`，主题沿用 `useTheme`。

## 页面接入

- 聊天、书籍详情、章节、统计、服务、设置、日志、题材、市场雷达、环境诊断等页面使用 App 的统一顶部栏。
- 写作模式继续显示“模式列表 / 新建模式 / 模式详情”，改用共享 tab 样式。
- 导入页面的章节、设定、同人、衍生等 tab 改用共享 tab 样式。
- 设置页移除通用卡片中的重复语言/主题控件，保留其他设置内容。
- 页面级返回、新建、刷新、保存按钮只在对应页面状态下显示，避免无效操作常驻。

## 视觉与响应式

- toolbar 高度、边框、背景、激活下划线、按钮圆角和间距由共享组件统一控制。
- 全局操作使用紧凑按钮，保留文字标签以支持可发现性和无障碍。
- 页面 tab 在窄屏下 `overflow-x-auto`，正文宽度不被导航挤压。
- 颜色通过现有主题 token 和 `useColors` 体系取值，不新增独立颜色系统。

## 错误处理

- 语言保存失败时保留当前界面语言，并使用现有页面提示机制显示错误。
- toolbar 操作不会吞掉页面自身的加载和错误状态。
- 组件接收空 tab/action 时仍渲染稳定的标题和全局操作区域。

## 测试

- 为 `PageToolbar` 添加渲染契约测试：标题、tab 激活态、全局操作、窄屏滚动类名。
- 为 App 添加路由到页面标题的单元测试。
- 更新写作模式和导入页面测试，确认原有 tab 标签和激活逻辑保持。
- 运行 Studio 全量测试、类型检查和构建。
- 使用内置浏览器逐页检查聊天、设置、写作模式、导入和服务配置的顶部导航及语言/主题切换。
