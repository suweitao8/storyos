# 短篇故事原创化改编设计

## 目标

让写作模式只负责提供可迁移的叙事节奏和创作机制，短篇故事必须在此基础上重新构造人物、地点、关系、因果链、核心规则、场景事件和结局，避免根据 B 站字幕或模式中的原始大纲 1:1 复刻。

模式仍然可以保存一份默认故事设定，用户可以查看和重新随机生成。默认设定生成完成后，同时保存一份明确的“原创化改编方案”，供后续短篇创作复用，不因进入创建页再次调用模型。

## 现状与问题

- `CraftProfile` 同时保存 `worldview`、`storyOutline`、视频节拍事件、反转和收束信息。
- `StorySeed` 虽然已经要求“不要复制”，但它的具体故事设定和原始分析内容仍会被送入短篇大纲/正文链路。
- `buildCraftGuide` 会把参考世界观、大纲、视频事件和原文范例注入写作提示词。原始剧情材料和范例句子与“不要复制”的文字约束同时出现，模型仍容易沿用原故事。
- 当前短篇流程没有一个可检查的“哪些内容必须改变、哪些内容只迁移功能”的中间层。

## 方案

### 1. 扩展缓存故事设定

在 `StorySeed` 增加可选字段 `originalizationPlan`，对应一个新的“原创化改编方案”板块。该字段保持 Markdown 文本，不引入第二套脆弱的 JSON 解析协议，旧的十板块 `story_seed.json` 继续可读。

改编方案必须包含：

- 允许继承的节奏功能：钩子、压力升级、信息释放、反转位置、高潮和余味的相对功能；
- 必须替换的维度：时代/地点、职业/身份、人物关系、核心规则或资源、关键道具、因果链、场景事件和结局；
- 本次改编的全新选择：新的故事空间、角色职责、利益关系、压力来源和代价；
- 节拍迁移表：把每个抽象节拍功能映射到全新的事件任务，不复述源故事事件；
- 禁止继承清单：专有名词、人物组合、连续事件顺序、独特对白/措辞、原作标志性道具和场景；
- 创作前自检：确认新故事不是“换名词”，而是重建了关系和因果链。

这个字段是可选的，以兼容已保存的旧模式。旧模式没有该字段时，短篇链路使用通用原创化契约和已有 `videoStory.originalizationRules`，不触发隐式模型请求。

### 2. 默认设定生成输出原创化方案

保留生成阶段读取原始 `worldview`、`storyOutline` 和节奏分析的能力，因为这一阶段的任务就是把参考内容加工成新的故事设定。但 `buildStorySeedPrompt` 必须明确区分两种输入：

- 参考资料只用于识别节奏、机制和情绪运动；
- 输出的故事设定必须是新的，原创化改编方案必须具体说明替换后的身份、空间、关系、因果链和结局。

生成输出增加“原创化改编方案”板块。随机重新生成时，上一版只用于保留有效机制和产生实质不同的替代方案，不能把上一版的事件顺序直接延续下去。

后台生成、详情页手动重新生成、短篇故事页手动重新生成都复用这个协议。缓存逻辑保持不变，不会因为打开故事创建页而重新生成。

### 3. 短篇写作只注入原创化后的创作契约

新增短篇专用的 craft guide 构建函数，短篇大纲和正文阶段使用它，不再直接使用完整参考模式注入：

- 保留结构、场景节奏、信息揭示、视角、情绪运动等抽象方法；
- 视频只保留节拍的相对位置、节拍类型、叙事功能和情绪效果；不注入 `event`、`evidence`、具体反转内容、收束事件或连续事件链；
- 鬼故事只保留恐惧机制、触发/限制/升级方法，要求重新设计超自然规则和场景；
- 注入缓存的 `originalizationPlan`，作为短篇大纲的硬约束和创作前检查表；
- 不注入参考作品的 `worldview`、`storyOutline`、视频 logline/outline/具体事件链或原文 `exemplars`。

短篇输入方向仍可以显示并编辑完整的默认故事设定，但大纲/正文模型必须把它视为待改编的创作基准：新稿至少改变故事空间、职业/身份、关系结构、核心因果链和结局中的关键代价。这样可以实现“学校的叙事机制迁移到写字楼、学生关系改造成上班族关系”，而不是只替换几个名词。

### 4. 长篇兼容范围

本次行为改变先限定在短篇故事生产链路，避免影响已有长篇章节的世界观和连续性输入。公共 `buildCraftGuide` 保留给现有长篇/写作者链路使用；短篇使用新的原创化专用 guide。后续如果长篇也需要同样的来源隔离，可以复用这层抽象化函数。

## 数据流

```text
参考字幕/文本
  -> CraftProfile（完整解析，供模式详情查看）
  -> StorySeed + originalizationPlan（后台缓存/主动重新生成）
  -> 短篇创建方向 + 短篇原创化 craft guide
  -> 短篇大纲：重新选择空间、身份、关系、因果链
  -> 短篇正文：只按新大纲写作，迁移节奏功能，不读取原始事件链
```

## 错误与兼容

- 旧 `story_seed.json` 缺少 `originalizationPlan` 时仍然通过 `isStorySeed` 校验。
- 旧种子序列化时不输出空的新增板块；新种子会在详情页和创建页显示该板块。
- 改编方案生成失败时保留上一份完整设定，短篇使用通用原创化规则；不会阻塞写作模式进入详情或短篇创建。
- 不新增每篇短篇的额外模型审查调用，避免把创建等待时间翻倍。原创性依靠输入隔离、结构化改编方案和大纲阶段硬约束保证。

## 验证

- Core：测试新板块解析/序列化、旧十板块种子兼容、短篇专用 guide 不包含原始世界观/故事大纲/视频具体事件/原文范例，且包含节奏功能和原创化约束。
- Core：测试故事种子提示明确要求原创化改编方案、空间/身份/因果链替换和禁止复制连续事件。
- Studio：测试后台和手动生成仍保存带改编方案的种子，缓存命中不调用模型。
- Studio：测试短篇运行链将短篇专用 guide 传给大纲/正文，并且不再传入原文 exemplars。
- 运行定向 Vitest、Core build、Studio typecheck；最后执行 `git diff --check`。
