# 写作模式源文件保留与详情重构设计

## 目标

让写作模式成为可重复使用的分析档案：创建模式时保留原始小说文件、B 站视频和字幕；详情页可以查看不同类型的分析内容；当用户不满意时，可以基于已保存源内容重新解析，而不会丢失源文件或旧结果。

## 设计

### 源资产生命周期

导入接口先在项目运行目录的 `craft-source-uploads/<asset-id>/` 创建临时资产。小说上传保存原始字节，同时保存解码后的分析文本；B 站导入保存下载后的视频、字幕 JSON、字幕 TXT 和分析文本。分析接口成功后，将临时目录原子移动到 `crafts/<craft-id>/source/`，并写入源资产元数据。分析失败时清理临时资产，不影响已有模式。

模式目录中的源资产约定为：

- `source/`：`novel.<ext>` 或 `video.mp4` 等原始输入；
- `subtitles.json` / `subtitles.txt`：B 站字幕（如果存在）；
- `analysis-input.txt`：本次解析使用的文本；
- `source-meta.json`：来源类型、原始名称、BVID、时长、字幕来源和文件清单。

重新解析读取 `analysis-input.txt` 和 `source-meta.json`，调用现有分析管线并覆盖 `craft_profile.json`，保留 `source/` 和字幕文件。没有源资产的历史模式显示“源文件未保存”，重新解析按钮不可用。

### API

- 扩展 `/craft/upload` 与 `/craft/bilibili/import` 响应，返回 `sourceAssetId`。
- 扩展 `/craft/analyze` 请求，接收 `sourceAssetId`，成功后归档源资产。
- 新增 `POST /crafts/:id/reparse`，基于已保存分析输入重新生成同一个模式。
- 新增 `GET /crafts/:id/source/:file`，只允许下载该模式元数据中登记的文件，并使用附件响应。
- 详情查询返回模式 profile 与源资产状态，旧客户端仍可使用 profile 字段。

### 详情页布局

详情页不再显示与导航栏重复的模式标题。顶部只保留来源类型、分析时间、模块数、范例数和操作按钮。内容分为五个可切换分区：概览、视频结构、拆文模块、范例、源文件。小说模式隐藏视频结构；没有对应内容的分区显示明确的空状态。源文件分区显示文件名、类型、大小、字幕状态和下载入口，并提供重新解析按钮。

### 错误处理

- 下载或转码失败时，不创建半成品模式，并返回可读错误。
- 重新解析失败时保留旧 `craft_profile.json`，接口返回错误；成功后再替换结果。
- 所有 craft id、asset id 和下载文件名都进行路径安全校验。
- 历史模式缺少源资产时仍可查看和删除，只禁用重新解析。

### 验证

- 单元测试覆盖源资产归档、重新解析所需输入、下载路径安全和详情分区选择。
- API 测试覆盖小说上传保留、分析归档、重新解析成功/失败和旧结果保留。
- B 站下载逻辑使用网络层与转码层的可替换依赖测试，不在测试中访问真实 B 站。
- 完成后运行 Studio 全量测试、core 构建、Studio 类型检查、客户端构建，并用浏览器验证详情页分区和重新解析按钮状态。
