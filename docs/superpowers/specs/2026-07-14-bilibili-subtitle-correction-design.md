# B 站字幕文字校正设计

## 背景

B 站写作模式导入流程会从公开 CC 字幕或 BCut 音频识别得到带时间轴的字幕，并直接将识别文本交给写作模式分析。自动识别可能产生同音字、形近字或语义明显不成立的文字错误，例如将“倒着烧香”识别成“盗箱”。

## 目标

- 在字幕提取完成后、写作模式分析前增加一次 AI 字幕校正。
- 只修正结合上下文可以判断为识别错误的文字，不改写内容、不扩写、不改变时间轴和字幕段落边界。
- 原始字幕必须保留，校正文本作为写作模式分析输入。
- 校正过程沿用项目关闭思考模式的 LLM 配置，只使用模型正文输出。
- 校正失败时回退原始字幕，使导入流程仍可继续，并向界面说明已回退。

## 方案

在 Studio 的 B 站导入接口中增加独立的字幕校正服务：

1. `importBilibiliSource` 完成字幕获取后，保留原始 `BilibiliSubtitleEntry[]`。
2. 导入接口使用当前项目的 pipeline LLM client/model 调用字幕校正服务。
3. 校正服务以带序号、时间轴和正文的字幕段落作为输入，要求模型返回严格 JSON 数组，每项只包含序号和校正后的 `content`。
4. 服务校验返回数量、序号、文本非空及时间轴边界，任何格式或完整性错误都抛出异常。
5. 接口成功时：
   - `subtitles.json` 保存原始字幕；
   - `subtitles.txt` 保存原始字幕文本，便于回看和追溯；
   - `analysis-input.txt` 保存校正后的带时间轴文本；
   - 返回校正后的预览和分析文本，并返回校正数量。
6. 接口失败时：记录警告，使用原始文本作为 `analysis-input.txt` 和返回分析文本，并返回回退标记及提示信息。

## LLM 约束

系统提示明确要求：

- 只修正 ASR/OCR 造成的错别字、同音字、形近字和明显不合语境的词；
- 不润色、不改语气、不补充缺失内容；
- 保留每个字幕段落的原顺序和边界；
- 没有把握时保持原文；
- 只输出 JSON，不输出思考过程、解释或 Markdown 代码块。

调用使用现有 `chatCompletion`，项目 provider 已将 `thinkingBudget` 统一限制为 0，因此不新增任何思考模式参数。

## 界面反馈

B 站导入期间在现有进度信息中显示“正在校正字幕文字”。导入结果中展示：

- 校正成功：`已完成字幕文字校正，修正 X 处`；
- 发生异常并回退：`字幕文字校正失败，已使用原始字幕`。

预览展示最终用于分析的字幕，原始字幕仍可从已保存的来源文件中追溯。

## 错误处理

- 模型返回非法 JSON、缺少段落、重复序号或空文本时视为校正失败。
- 校正失败不影响 B 站导入和写作模式创建。
- 原始字幕获取失败仍按现有流程报错，不进入校正阶段。
- 不记录字幕正文到普通日志，只记录阶段和错误摘要。

## 测试

- 校正结果能保持原始时间轴和段落顺序，只替换正文。
- 合法 JSON 能正确解析并统计变更数量。
- 非法、缺失或重复段落会被拒绝。
- 校正调用失败会回退到原始文本。
- 导入接口保存原始字幕文件，同时将校正文本作为分析输入。
- UI 显示校正阶段和成功/回退状态。
