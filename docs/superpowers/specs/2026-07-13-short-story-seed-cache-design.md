# 短片故事设定缓存与全宽创建页设计

## 目标

让每个写作模式拥有一份可复用的短片故事完整设定。创建故事页进入或重新挂载时，优先读取已保存设定；只有当前写作模式还没有设定，或用户主动点击“重新随机生成”时，才调用模型生成。与此同时，创建页内容和滚动容器取消当前最大宽度限制，填满可用页面宽度。

## 已确认的根因

- `StoryCreationPanel` 的短篇初始化 effect 在面板挂载、会话变化或写作模式变化时都会调用 `onGenerateSeed`，没有检查当前写作模式是否已有设定。
- `CraftProfile` 和 `CraftMeta` 没有故事设定字段，写作模式目录也没有可供创建页恢复的缓存。
- `StoryCreationPanel` 根节点使用 `max-w-[1440px]` 并居中，自己的纵向滚动条因此停留在受限容器右侧，而不是页面最右侧。

## 方案

### 持久化

- 在 `CraftProfile`/`CraftMeta` 暴露可选的 `storySeed?: StorySeed`。
- 在每个写作模式目录下单独保存 `story_seed.json`，避免重新解析参考内容时覆盖用户已经生成的设定。
- `PipelineRunner.loadCraft` 合并该文件，`listCrafts` 将其带入写作模式列表；没有缓存的旧模式保持兼容。
- 增加 `saveCraftStorySeed`，只更新 `story_seed.json`。

### API

- 增加 `PUT /api/v1/crafts/:id/story-seed`。
- 请求体为 `{ "storySeed": <完整十段设定> }`，服务端校验十个字段均为非空字符串后保存。
- 保存成功返回 `{ storySeed }`；找不到写作模式或数据非法时返回结构化错误。

### 创建页状态

- 选择到已有 `storySeed` 的写作模式时，直接恢复十段可编辑预览，并将状态设为 ready，不启动生成请求。
- 选择没有缓存的写作模式时保持首次自动生成行为；无写作模式时也保持当前会话的首次生成行为，因为没有可持久化的写作模式归属。
- 用户点击“重新随机生成”时显式绕过缓存，生成完成后覆盖当前写作模式的 `story_seed.json`。
- 当前页面内的编辑仍立即用于创建；自动生成的完整设定会持久化，避免在输入框每次按键时发送保存请求。
- 保存失败不丢弃已经生成的预览，页面提示保存失败，用户仍可继续创建或稍后重新生成。

### 布局

- `StoryCreationPanel` 根节点移除 `mx-auto` 和 `max-w-[1440px]`，保留 `w-full`、内边距与纵向滚动。
- 两列布局继续使用 `minmax(0, ...)`，保证长文本不会撑破容器。

## 验收标准

1. 已有缓存的写作模式进入创建页时，生成函数调用次数为零，预览内容等于缓存内容。
2. 没有缓存的写作模式首次进入仍可生成；生成完成后服务端存在对应 `story_seed.json`。
3. 点击“重新随机生成”会再次调用生成函数并替换缓存。
4. 服务端拒绝缺字段或空字段的设定，不覆盖旧缓存。
5. 既有旧写作模式、故事创建、故事流式预览测试继续通过。
6. 创建页不再包含 `max-w-[1440px]`，滚动容器宽度为可用页面宽度。
