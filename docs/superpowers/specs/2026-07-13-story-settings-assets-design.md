# 故事设定与故事资产拆分设计

## 背景

当前故事创建完成后，故事内容、章节阅读和对话调整共用一个双栏界面。随着世界观、大纲、角色、场景和道具信息增加，阅读区与调整区互相挤压，后续也没有稳定的资产引用入口。

本次设计将工作区拆成制作阶段，并借鉴 `D:\Github\animcg` 的角色、场景、道具资产结构，为后续剧本、分镜和视频制作提供稳定的参考图资产。

## 已确认约束

- 使用顶部阶段导航，当前选择为方案 A。
- 资产按当前故事独立保存，不做跨故事共享。
- 故事生成后自动提取文字资产和图片提示词。
- 默认不调用图片模型。
- 支持逐个生成和一键生成全部缺失图片。
- 每个角色、场景、道具第一版只生成一张主参考图。
- 后续剧本、分镜和视频统一引用资产，不重复临时描述同一资产。

## 方案

### 阶段导航

顶部统一显示：

1. 故事设定
2. 故事资产
3. 对话调整
4. 剧本
5. 分镜
6. 视频

本次实现完整启用前 3 个阶段。剧本、分镜和视频保留统一导航位置，若当前功能尚未可用则显示未开放状态，不伪造可用内容。

### 故事设定

故事设定使用全宽阅读区，不再和聊天同时占据同一屏幕。内容分组为：

- 世界观与规则
- 故事大纲
- 主线冲突与节奏
- 角色关系摘要
- 平台、篇幅和生成参数

设定页面优先读取现有故事文件和章节内容，不改变已有故事文件格式。页面提供“进入对话调整”入口；对话调整完成后刷新设定页面，避免在同一界面同时显示长文本和聊天记录。

### 对话调整

对话调整使用全宽聊天界面。原有故事创建后的聊天能力继续保留，但不再在聊天页面左侧常驻故事全文。故事内容通过“故事设定”阶段查看，避免双栏造成信息拥挤。

### 故事资产

故事资产使用全宽卡片工作区：

- 顶部操作：提取资产、生成全部缺失图片、刷新
- 类型筛选：角色、场景、道具
- 卡片内容：名称、简略描述、详细设定、图片提示词、参考图状态
- 卡片操作：编辑、生成参考图、重新生成
- 没有图片时显示稳定的空状态，不生成默认图片
- 生成失败时保留提示词和错误状态，支持重新生成

角色、场景和道具的字段参考 `animcg`：

- 角色：身份、年龄、性别/物种、身高、外貌、服装、气质、音色描述、图片提示词
- 场景：空间名称、空间类型、布局、时代/风格、光线、氛围、图片提示词
- 道具：用途、外观、材质、关键细节、剧情作用、图片提示词

### 资产提取与图片生成

资产提取由故事设定、故事大纲和正文共同作为输入。提取流程只调用文本模型，输出结构化文字资产和图片提示词；同类型同名资产去重，已有资产更新文字字段时保留已有图片。

图片生成只允许通过明确操作触发：

- 单卡“生成参考图”
- 顶部“生成全部缺失图片”

批量操作只处理没有 ready 图片的资产，不重复消耗已有图片。图片生成使用现有图片服务配置，未配置或连接失败时只返回可理解的错误，不影响文字资产保存。

## 数据模型

增加故事级资产清单，建议保存为：

- 长篇：`books/<bookId>/assets/manifest.json`
- 短篇：`shorts/<storyId>/assets/manifest.json`
- 图片：同一故事目录下的 `assets/images/<assetId>.<ext>`

核心结构：

```ts
type StoryAssetKind = "character" | "scene" | "prop";
type StoryAssetImageStatus = "missing" | "generating" | "ready" | "error";

interface StoryAsset {
  id: string;
  kind: StoryAssetKind;
  name: string;
  summary: string;
  details: Record<string, string>;
  imagePrompt: string;
  sourceRefs: string[];
  image: {
    status: StoryAssetImageStatus;
    path?: string;
    error?: string;
    generatedAt?: string;
  };
  createdAt: string;
  updatedAt: string;
}

interface StoryAssetManifest {
  version: 1;
  storyId: string;
  updatedAt: string;
  assets: StoryAsset[];
}
```

资产清单是故事的持久化边界，后续剧本、分镜和视频只引用 `asset.id` 与图片路径，不直接复制整段提示词。

## API 边界

按故事类型解析到 `books/<id>` 或 `shorts/<id>`，不接受任意文件路径：

- `GET /api/v1/stories/:kind/:id/assets`
- `POST /api/v1/stories/:kind/:id/assets/extract`
- `PATCH /api/v1/stories/:kind/:id/assets/:assetId`
- `POST /api/v1/stories/:kind/:id/assets/:assetId/generate-image`
- `POST /api/v1/stories/:kind/:id/assets/generate-missing-images`
- `GET /api/v1/stories/:kind/:id/assets/images/:assetId`

所有 id 和文件名都必须经过现有路径安全校验。生成接口只接受清单中已存在的资产，不能通过请求体写入任意输出路径。

## 状态与错误处理

- 提取中：显示文本提取进度，图片按钮保持禁用或显示处理中。
- 图片生成中：单卡显示 generating，批量生成按资产逐个更新状态。
- 图片生成成功：保存图片路径和 generatedAt，状态为 ready。
- 图片生成失败：保存 error，不删除 prompt，允许单卡重试。
- 页面刷新：从 manifest 恢复所有状态，不依赖前端内存。
- 故事不存在或资产不存在：返回明确的 404，不创建隐式目录。

## 验证计划

- 核心单元测试：角色/场景/道具归一化、同名去重、已有图片保留、清单读写。
- API 测试：提取、单卡生成、批量生成、失败状态、路径穿越防护。
- Studio 页面测试：阶段导航高亮、设定/资产页面分离、资产筛选、空状态、生成按钮状态。
- 运行态验证：默认提取不调用图片模型；点击单卡和批量按钮才发起图片生成请求；刷新后资产状态仍存在。

## 非目标

- 本次不实现完整剧本、分镜和视频生成流程，只提供统一导航位置和可扩展边界。
- 本次不建立跨故事全局资产库。
- 本次不自动生成多视角、多表情或多状态图片。
- 本次不改变已有小说、短篇正文和模式数据格式。
