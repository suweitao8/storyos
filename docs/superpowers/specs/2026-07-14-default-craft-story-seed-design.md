# 写作模式默认故事设定设计

## 目标

每个写作模式保存一份自己的默认故事设定。创建故事时优先直接读取这份缓存，不因进入短篇故事页面而自动调用模型；用户可以在写作模式详情页或短篇故事页面主动重新随机生成并覆盖设定。

## 现状与问题

- `PipelineRunner` 已经通过 `story_seed.json` 保存 `StorySeed`，并在 `loadCraft`、`listCrafts` 中返回 `storySeed`。
- `StoryCreationPanel` 当前在短篇模式没有缓存设定时，进入页面就自动调用 `/crafts/:id/story-direction/stream`，导致每次进入都等待模型。
- 写作模式创建完成后没有后台生成默认设定，因此首次创建故事经常触发这次等待。
- 写作模式详情页没有专门的故事设定标签和编辑/重新生成入口。

## 方案

### 数据与生命周期

继续使用 `crafts/<craftId>/story_seed.json` 作为唯一设定内容来源，在 `CraftMeta` 增加可选状态字段：

- `storySeedStatus`: `pending`、`ready`、`error`
- `storySeedError`: 最近一次后台生成失败信息

保存设定时同时将状态写成 `ready`；后台开始生成时写成 `pending`，失败时写成 `error`，不删除旧设定。旧模式没有这些字段时按照已有 `storySeed` 推断为 ready/未生成。

### 后台预生成

- 小说/视频模式分析完成后启动一次后台默认设定生成；分析接口仍然先返回，不能把设定生成加入用户等待链。
- B 站异步导入在模式解析完成后继续执行默认设定生成，前端显示“正在生成默认故事设定”，完成后模式才显示完整就绪状态。
- 生成任务使用现有 `buildStorySeedPrompt`、`chatCompletion`、`parseStorySeed`，不启用思考模式；已有设定时直接跳过。
- 任务失败只记录状态并保留模式，用户可在详情页或短篇页面主动重试。

### 写作模式详情页

新增独立“故事设定”标签：

- 有缓存时直接展示十个设定板块，不发起模型请求。
- 无缓存或状态 error 时显示生成按钮和错误信息。
- pending 时显示进度状态，不重复提交任务。
- “重新随机生成”使用现有流式接口，完成后保存并立即更新详情页；生成期间右侧持续展示原始输出。

### 故事创建页

- 短篇故事页只读取所选模式的 `storySeed`，不再在 `useEffect` 中自动生成。
- 有缓存时直接填入故事设定；没有缓存时保持空闲状态，由用户点击“随机生成”触发。
- 长篇故事页优先把缓存设定作为默认故事方向；原有“重新生成方向”按钮仍可由用户主动调用模型。
- 没有选择写作模式时也不自动生成，避免进入页面产生隐式模型请求。

## 错误与兼容

- `story_seed.json` 保持原格式，已有模式和已有故事不需要迁移。
- 生成失败不会覆盖上一份有效设定。
- 原有故事设定流式接口和保存接口继续保留；新增状态字段为可选，旧客户端可忽略。

## 验证

- Core 测试验证设定保存会更新状态、旧模式兼容、失败状态保留。
- Studio API 测试验证分析完成后启动后台预生成、已有设定跳过、流式接口命中缓存不调用模型。
- Studio 页面状态测试验证无缓存不自动生成、有缓存直接 ready、手动重新生成仍可用。
- 运行 Core/Studio 定向测试、类型检查和前端/后端构建。
