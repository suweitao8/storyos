# 题材页面布局优化

## 目标

- 移除内容区重复的“题材”页面标题，仅保留 Studio 顶部工具栏标题。
- 将页面组织为左侧题材列表、右侧题材详情的双栏布局。
- 保留题材查看、创建、编辑、复制和删除行为，不改变题材规则数据流。
- 在窄屏下自动切换为上下布局，避免横向溢出。

## 方案

1. `GenreManager` 不再渲染重复的 `h1`，创建按钮保留在内容区顶部右侧。
2. 左栏显示“题材列表”和当前语言可用题材数量，题材条目保持可点击选中状态。
3. 右栏保留编辑、复制、删除操作和题材规则内容，未选中时显示选择提示。
4. App 为题材页使用更宽的内容容器，桌面端使用固定列表宽度加自适应详情宽度，移动端改为单列。

## 验证

- 运行题材相关 Studio 测试、类型检查和客户端构建。
- 通过 Studio 运行时检查桌面与移动端布局，确认无横向溢出、标题不重复、列表与详情均可操作。
